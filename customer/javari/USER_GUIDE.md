## CREDENTIAL VAULT

### What is the Credential Vault?

The most secure way to store API keys, passwords, and tokens. Javari uses:
- **AES-256-GCM encryption** (same as banks)
- **Zero-knowledge architecture** (we can't see your credentials)
- **Automatic rotation** (optional, prevents expiration)
- **Complete audit trail** (know exactly when credentials are accessed)

### Adding Credentials

**Quick Add:**
1. **Click** "Credentials" in sidebar
2. **Click** "Add Credential"
3. **Fill in details:**
   - Service name (e.g., "GitHub")
   - Credential name (e.g., "Personal Access Token")
   - Actual credential value
   - Expiration date (optional)
4. **Save** - Encrypted immediately

**Supported Services (with auto-rotation):**
- GitHub personal access tokens
- Vercel API tokens
- Stripe API keys
- OpenAI API keys
- Supabase service keys
- Custom services

### Using Credentials in Conversations

**Automatic Injection:**  
When you work on a project that needs credentials, Javari automatically uses them (after asking permission once).

**Example:**
```
You: Deploy my website to Vercel

Javari: I'll use your Vercel credentials to deploy.
[Deploys using stored Vercel token]
‚úÖ Deployed successfully to https://your-site.vercel.app
```

**Manual Reference:**
```
You: What's my GitHub token?

Javari: For security, I won't display the full token. 
I can tell you:
- Service: GitHub
- Name: Personal Access Token
- Status: Active
- Expires: Never
- Last used: 2 days ago

Would you like me to test if it's still working?
```

### Credential Security

**Automatic Rotation (Optional):**
Enable auto-rotation for supported services:
1. **Edit** credential
2. **Toggle** "Auto-rotate"
3. **Choose** frequency: 30, 60, or 90 days
4. Javari handles rotation automatically
5. **Grace period:** Old credential works for 7 days

**Audit Log:**  
See every time a credential is accessed:
- Date and time
- Which conversation used it
- What action was performed
- Success or failure

**Access from:**  
Credentials ‚Üí [Credential Name] ‚Üí "Audit Log"

## TROUBLESHOOTING

### Common Issues

#### **"Out of Credits" Error**

**Solution:**
1. Check your current plan and usage
2. Upgrade to higher tier if needed
3. Purchase additional credit packs
4. Wait until next billing cycle (credits refresh)

**Prevent in future:**
- Set up credit alerts (Settings ‚Üí Notifications)
- Track usage in dashboard
- Use credit-efficient practices above

#### **Javari Doesn't Have Context**

**Problem:** Javari seems to forget previous discussion

**Solutions:**
1. ‚úÖ Check if you're in the same conversation
2. ‚úÖ Look for "conversation continued" notice
3. ‚úÖ Upload relevant docs to Knowledge Base
4. ‚úÖ Use project to group related chats

#### **Credential Not Found**

**Problem:** "I don't have access to that credential"

**Solutions:**
1. Add credential to vault
2. Make sure credential is in current project
3. Check credential hasn't expired
4. Verify spelling of credential name

#### **Slow Responses**

**Possible causes:**
- Complex request requiring multiple AI calls
- Large document being processed
- High platform load

**Solutions:**
- Wait patiently (usually resolves in 30-60 seconds)
- Refresh page if longer than 2 minutes
- Check status page: status.javariai.com
- Contact support if persistent

#### **Can't Find Previous Conversation**

**Solutions:**
1. Use search bar with keywords from conversation
2. Check if accidentally deleted
3. Filter by date range
4. Look in specific project
